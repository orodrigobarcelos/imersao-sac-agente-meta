# Ad Rules Engine

Updated: Feb 4, 2026The ad rules engine is a central rule management service that helps you easily, efficiently, and intelligently manage ads. Without it, you must query the Marketing API to monitor an ad’s performance and manually take actions on certain conditions.Since we can express most conditions as logical expressions, we can automate management in two ways: [Schedule-based rules](https://developers.facebook.com/documentation/ads-commerce/marketing-api/ad-rules/guides/scheduled-based-rules) or [Trigger-based rules](https://developers.facebook.com/documentation/ads-commerce/marketing-api/ad-rules/guides/trigger-based-rules).

## Ad Rules Objects

Ad rules are standalone objects created and stored in the ad rules library and minimally contain a `name`, an [`evaluation_spec`](https://developers.facebook.com/documentation/ads-commerce/marketing-api/ad-rules/overview/evaluation-spec) and an [`execution_spec`](https://developers.facebook.com/documentation/ads-commerce/marketing-api/ad-rules/overview/execution-spec).

### Basic structure


```
curl -X POST \
  -F 'name=Rule 1' \
  -F 'evaluation_spec={
    ...
  }' \
  -F 'execution_spec={
    ...
  }' \
  -F "access_token=<ACCESS_TOKEN>" \
https://graph.facebook.com/v25.0/act_<AD_ACCOUNT_ID>/adrules_library
```


## Options


### [Trigger-based rules](https://developers.facebook.com/documentation/ads-commerce/marketing-api/ad-rules/guides/trigger-based-rules)

Monitor the state of your ads in real-time. A trigger-based rule is evaluated as soon as its pertinent ad objects’ metadata or [Ad Insights](https://developers.facebook.com/documentation/ads-commerce/marketing-api/reference/adgroup/insights) data are changed.

### [Schedule-based rules](https://developers.facebook.com/documentation/ads-commerce/marketing-api/ad-rules/guides/scheduled-based-rules)

Monitor the state of your ads by checking them at a set time interval to see if they meet the `evaluation_spec` criteria.

## Components


### [Evaluation spec](https://developers.facebook.com/documentation/ads-commerce/marketing-api/ad-rules/overview/evaluation-spec)

The main purpose of the `evaluation_spec` of a rule is to determine the objects upon which the rule should execute its action.

### [Execution Spec](https://developers.facebook.com/documentation/ads-commerce/marketing-api/ad-rules/overview/execution-spec)

The `execution_spec` of a rule determines the action that applies to all objects that pass evaluation.

### Status

The `status` of a rule determines whether the rule should be running.To temporarily turn off a rule, set its status to `DISABLED`. To reactivate a rule, set its status to `ENABLED`.
